About PharmAI

PharmAI is an end-to-end intelligent manufacturing platform built for the pharmaceutical industry. It combines machine learning, deep learning, and explainable AI to predict batch quality, detect anomalies, forecast energy usage, and generate automated corrective action alerts — all before a batch fails.

01 — The Problem

Pharmaceutical manufacturing is high-stakes. A single batch failure costs crores in wasted materials, regulatory delays, and downtime. Traditional quality control is reactive — it only catches failures after they happen. There is no early warning, no explainability, and no way for operators to act before it is too late.

PROBLEM 01
Batch Failures Found Too Late

Quality checks happen after the batch completes. By then, materials, time, and energy are already wasted with no way to recover.

PROBLEM 02
No Anomaly Detection

Equipment degradation and process deviations go unnoticed until they cause a failure. There is no per-phase risk scoring or early warning system.

PROBLEM 03
Operators Can't Read ML Output

Raw model predictions mean nothing to a floor operator. They need clear, actionable instructions in plain language, not numbers and probabilities.

PROBLEM 04
Energy Waste is Invisible

Drying and compression consume significant energy. Without forecasting, inefficient settings go unchallenged and carbon footprint grows silently.

PROBLEM 05
Maintenance is Reactive

Equipment is serviced only after it breaks. No predictive system exists to flag a phase heading for failure before costly downtime occurs.

PROBLEM 06
No Real-Time Visibility

Supervisors have no live view across all manufacturing phases simultaneously — no dashboard, no live alerts, no drill-down capability.

02 — What We Built

PharmAI solves every one of these problems with four purpose-built AI models working together as a single platform, connected through a FastAPI backend and a live multi-page monitoring interface.

🤖
Batch Quality Predictor

An ensemble of XGBoost, Random Forest, and Gradient Boosting predicts all 6 quality targets — Hardness, Friability, Content Uniformity, Dissolution Rate, Tablet Weight, and Disintegration Time — from 8 process parameters before the batch completes. Operators receive a PASS or FAIL verdict in milliseconds.

XGBoost 40%Random Forest 35%Gradient Boost 25%6 Quality Targets>90% Accuracy
⏱️
LSTM Energy Forecasting

A 2-layer LSTM reads 15-minute sensor windows across 5 channels — power, vibration, temperature, motor speed, and pressure — to predict next-minute energy consumption. Supervisors can catch dangerous buildup patterns and adjust parameters before energy spikes cause process deviations.

64-unit LSTM15-min window5 sensor channelsEarlyStopping
🔬
Autoencoder Anomaly Detection

An Autoencoder trained on normal batch data learns what normal looks like. Any input with reconstruction error above μ+2σ is flagged as an anomaly. Risk scores are computed per manufacturing phase — giving a live view of which phase is heading for failure, up to 24 hours in advance.

Per-phase risk6→2→6 bottleneckμ+2σ thresholdPredictive maintenance
🧠
Agentic AI Alert System

The Agentic AI layer reads SHAP feature importance, anomaly flags, and maintenance risk scores — then generates plain English corrective action alerts ranked by severity. Instead of raw numbers, the operator sees: "Reduce Drying Temperature — over-drying detected. Check Moisture Content."

SHAP-poweredPlain EnglishCritical / Warning / InfoCorrective actions
03 — How It Works
01
Operator Sets Parameters

The floor operator enters 8 process parameters for the current batch — Compression Force, Machine Speed, Drying Temperature, Drying Time, Granulation Time, Binder Amount, Moisture Content, and Lubricant Concentration — using the Predictor page sliders.

8 inputs → Predictor page
02
Feature Engineering

PharmAI automatically derives 4 physics-based interaction features from the 8 inputs — Drying Efficiency, Granulation Intensity, Compression Intensity, and Moisture–Lubricant Ratio — expanding the input space to 12 features before the models run.

8 raw → 12 engineered features
03
Ensemble Model Predicts Quality

All three models run in parallel and their outputs are combined using weighted voting. Six quality values are predicted simultaneously. Each is checked against its pharmaceutical specification to issue a PASS or FAIL verdict instantly.

POST /predict → FastAPI → 6 predictions in <200ms
04
Agentic AI Generates Alerts

If any target fails, the Agentic AI identifies the root cause using SHAP importance values and generates a specific corrective action message. The operator sees a clear instruction, not a raw number — and can act on it immediately.

SHAP values → alert severity → plain English action
05
Dashboard Monitors Everything Live

In parallel, the Dashboard continuously polls FastAPI for live KPIs, LSTM power predictions, and Autoencoder maintenance risk per phase. Supervisors see a real-time view across all 8 manufacturing phases with automatic anomaly alerts.

GET /dashboard/metrics · /lstm/predict · /maintenance
